Kontrolery w projekcie REST API stworzonym w ASP.NET Core pełnią kluczową rolę w obsłudze żądań HTTP i są centralnym punktem, w którym logika aplikacji łączy się z protokołem HTTP. Oto główne zadania kontrolerów w takim projekcie

Kontrolery przyjmują żądania HTTP od klientów (np. przeglądarek internetowych, aplikacji mobilnych, innych usług) i decydują, jak na nie reagować. Każda metoda w kontrolerze jest zwykle powiązana z konkretnym typem żądania HTTP (GET, POST, PUT, DELETE itp.) i określonym endpointem (ścieżką URL). Kontrolery często odpowiadają za walidację danych wejściowych przekazanych w żądaniach, np. parametrów zapytania, ciał żądań JSON, itp. ASP.NET Core oferuje funkcje walidacji modeli, które ułatwiają sprawdzanie poprawności danych wejściowych.
